# 프로젝트 작업 규칙

## 파일 구조
- `helper1.html`, `helper2.html` — 실제 프로그램 본체 (canonical)
- `생기부 작성 도우미1.html`, `생기부 작성 도우미2.html` — 한글 URL 보존용 리다이렉트 (helper1/2로 자동 이동)
- `index1.html`, `index2.html` — 짧은 URL 리다이렉트 (helper1/2로 자동 이동)
- 푸터 날짜는 `document.lastModified`로 자동 갱신됨 (수동 수정 불필요)
- 푸터 버전(`v1.0`)은 HTML에 하드코딩 — **아래 규칙대로 수동 증가**

## 버전 자동 증가 규칙 (SemVer)

수정 작업을 할 때마다 변경 규모를 판단하여 `helper1.html`, `helper2.html` 두 파일의 푸터 버전을 자동으로 올린다. 위치:

```html
<div class="footer-sub">마지막 업데이트 <span id="lastUpdate"></span> · vX.Y.Z · 계속 다듬어 나가는 중입니다</div>
```

판단 기준:

| 변경 종류 | 예시 | 버전 변화 |
|---|---|---|
| **메이저** (구조 개편, 사용 흐름이 크게 바뀜) | 프롬프트 구조 전면 개편, API 구조 재설계, 핵심 기능 영역 교체 | `1.x.x` → `2.0.0` |
| **마이너** (새 기능 추가, 기존 흐름 유지) | 새 항목/옵션 추가, 학생 수준 같은 신규 기능, 멀티 프로바이더 지원 | `1.0.x` → `1.1.0` |
| **패치** (버그 수정, 작은 다듬기) | 오타 수정, 색상/문구 조정, 작은 버그, 푸터 텍스트 변경 | `1.0.0` → `1.0.1` |

## 버전 적용 방법
1. 수정 작업 직전에 현재 버전을 두 파일에서 확인
2. 변경 규모를 판단하고 적절히 증가
3. 두 파일의 버전 표기를 동시에 같은 값으로 업데이트 (두 프로그램은 항상 같은 버전 유지)
4. 변경 내용과 함께 한 번에 커밋

## 배포 흐름
- 작업은 워크트리 (`claude/modest-jang` 브랜치)에서 진행
- 완료 후 `main`에 fast-forward 머지 → push → GitHub Pages 자동 배포

## 사용자 의사소통
- 한국어로 응답
- 큰 변경 전에는 먼저 의견을 제시하고 확인받기
- 코드 수정 후에는 무엇을 수정했는지 위치와 함께 명확히 알리기
